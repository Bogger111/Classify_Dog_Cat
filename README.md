# 猫狗图像分类

这是一个用 PyTorch 做的猫狗二分类项目。它从本地图片读入数据，训练一个自定义 ResNet18 风格的卷积网络，然后输出训练曲线、混淆矩阵和样例预测图。项目不只是跑出一个准确率，也把模型到底学得稳不稳、错在哪里用图展示出来。

## 项目内容

- `Resnet.py`：自定义 ResNet18 风格模型，包含残差块、BatchNorm、ReLU、AdaptiveAvgPool 和 Dropout。
- `dataloader.py`：读取猫狗图片，按文件名生成标签，并做训练集和验证集划分。
- `trainer.py`：训练、验证、保存 checkpoint，并在训练结束后生成结果图。
- `plot_results.py`：从 checkpoint 重新生成可视化结果。
- `app.py`：一个 Streamlit 上传图片识别的小界面。
- `config.yaml`：训练参数集中配置。

## 数据和训练设置

数据来自猫狗图片二分类任务，文件名以 `cat` 和 `dog` 开头来区分类别：

- 类别 0：猫
- 类别 1：狗

由于Kaggle自带的测试集缺少标签，当前配置会把 `data/train` 里的图片按 8:2 切成训练集和验证集。训练集使用随机裁剪、随机水平翻转和归一化进行数据增强，验证集使用中心裁剪和归一化提高验证效率。

主要训练参数：

- 模型：自定义 `MyResNet18`
- 训练轮数：50
- Batch size：16
- 优化器：Adam
- 学习率：0.001
- 权重衰减：0.0001
- 损失函数：CrossEntropyLoss
- 学习率调度：StepLR，每 20 轮衰减一次，gamma 为 0.8
- 混合精度：CUDA 可用时开启 AMP

## 训练结果

### 准确率曲线

![Train vs Val Accuracy](outputs/accuracy_curve.png)

准确率提升得比较快。前 10 轮模型从还不太会分，逐步涨到 85% 以上；大约第 20 轮后，训练准确率和验证准确率基本都进入 93% 到 96% 区间。后半段两条线贴得比较近，说明模型没有明显只记住训练集。

验证准确率中间有几次抖动，比如第 8 轮附近掉得比较明显，后面也有少量单点回落。这种现象在图像分类训练里比较常见，通常和随机增强、验证样本难度、batch 分布有关。整体看，模型最后稳定在 95% 左右，已经能比较可靠地区分猫和狗。

### 损失曲线

![Train vs Val Loss](outputs/loss_curve.png)

损失曲线和准确率曲线是对应的：前期下降很快，说明模型确实学到了有效特征；20 轮后下降变慢，进入比较平稳的收敛阶段。

验证损失在第 8 轮附近有一次尖峰，后面第 27 轮、第 41 轮附近也有小波动。不过这些波动没有持续扩大，训练损失和验证损失长期保持在接近的范围内，所以更像是训练过程中的正常抖动，而不是明显过拟合。

### 混淆矩阵

![Confusion Matrix](outputs/confusion_matrix.png)

混淆矩阵按行归一化后，可以更直观看出每个类别的表现：

- 真实为猫的图片，有 97% 被预测成猫，约 3% 被误判成狗。
- 真实为狗的图片，有 93% 被预测成狗，约 7% 被误判成猫。

也就是说，模型对猫的识别略稳一点，对狗的漏判相对多一些。后续如果继续优化，可以重点看被误判成猫的狗图：比如小狗、侧脸、遮挡、光线暗、背景复杂这些情况，可能会让模型更容易犹豫。

### 样例预测

![Sample Predictions](outputs/sample_predictions.png)

样例预测里，`T` 表示真实类别，`P` 表示模型预测类别。图中这批样本都预测正确，说明模型不只是指标好看，在实际图片上也能抓到比较稳定的猫狗特征。



## 结果小结

这次训练的模型已经进入比较可用的状态：

- 收敛速度正常，20 轮后进入稳定阶段。
- 训练集和验证集表现接近，没有明显过拟合迹象。
- 验证准确率稳定在 95% 左右。
- 猫类识别更稳定，狗类还有一点提升空间。



## 输出文件

- `outputs/accuracy_curve.png`：训练集和验证集准确率曲线
- `outputs/loss_curve.png`：训练集和验证集损失曲线
- `outputs/confusion_matrix.png`：验证集混淆矩阵
- `outputs/sample_predictions.png`：样例预测展示
- `checkpoints/Adam_lr0.001_CrossEntropyLoss_StepLR_best.pth`：验证集表现最好的模型
- `checkpoints/Adam_lr0.001_CrossEntropyLoss_StepLR_last.pth`：最后一轮模型

# Classify_Dog_Cat

## 项目简介

本项目是一个基于 PyTorch 的猫狗图像分类项目，使用自定义 ResNet18 网络结构完成二分类任务。

项目主要目的是熟悉深度学习图像分类任务的完整流程，包括数据读取、数据增强、模型搭建、训练验证、模型保存和结果可视化。

## 项目特点

- 使用 PyTorch 搭建训练流程
- 实现自定义 ResNet18 模型
- 支持训练集与验证集划分
- 支持模型 checkpoint 保存
- 支持训练曲线、混淆矩阵和预测样例可视化
- 支持通过配置文件管理训练参数

## 技术栈

- Python
- PyTorch
- Torchvision
- NumPy
- Matplotlib
- scikit-learn
- tqdm
- YAML

## 模型结构

本项目使用自定义实现的 ResNet18 结构，包含 BasicBlock、残差连接、卷积层、BatchNorm、ReLU 和全连接分类层。

相比直接调用 torchvision.models，本项目更注重对 ResNet 网络结构和训练流程的理解。

## 数据集

数据集来源于 Kaggle Dogs vs Cats 图像分类任务。

任务目标是根据输入图片判断其类别为猫或狗。

## 训练流程

主要训练流程包括：

1. 读取图像数据
2. 数据预处理与增强
3. 构建训练集与验证集
4. 初始化模型、损失函数、优化器和学习率调度器
5. 训练模型并记录 loss 和 accuracy
6. 保存验证集表现最好的模型
7. 绘制训练曲线和混淆矩阵

## 当前结果

目前模型在验证集上取得了约 96% 左右的准确率。

该结果说明模型已经能够较好地区分猫狗图像，但仍存在进一步优化空间，例如：

- 使用更强的数据增强
- 调整学习率策略
- 尝试预训练模型
- 增加错误样本分析
- 优化模型泛化能力

## 项目收获

通过本项目，我主要熟悉了：

- PyTorch 工程项目的基本组织方式
- CNN 和 ResNet 的基本原理
- 图像分类任务的完整训练流程
- 训练日志、验证指标和可视化分析方法
- 深度学习项目中配置文件、checkpoint 和结果管理的基本方法
